Сведения о команде Scribus (в порядке присоединения)

Franz Schmid (fschmid): «наш Линус». Автор и главный программист; разработчик программного обеспечения для производственной компании.

Peter Linnell (mrdocs): начал с тестирования версии 0.3.6, взял на себя написание документации для версии 0.5.0; тестировщик, сборщик RPM-пакетов, первоначальный веб-мастер сайта Scribus; работает консультантом по допечатной подготовке и ИТ для издательских компаний и компаний, занимающихся электронной коммерцией.

Craig Bradney (MrB): присоединился на версии 1.1.1; занимается программированием, тестированием, настройкой docs.scribus.net и bugs.scribus.net, администрирует IRC-каналы на freenode.net; работает менеджером по ИТ в международной компании, занимающейся электронной коммерцией.

Petr Vaněk: начал с версии 1.1.4; специалист по подключаемым модулям: создал несколько модулей для Scribus и написал руководство по созданию таких модулей; по профессии — инженер-программист.

Riku Leino (Tsoots): влился в команду на версии 1.1.5; разработал подключаемый модуль шаблонов документов и инструменты импорта OpenOffice.org/HTML для версии 1.2.2; менеджер по ИТ в крупной некоммерческой организации в Финляндии.

Andreas Vox — читается как [fox] — (avox): начал с версии 1.3.0; разработчик первоначальной версию программы для Mac OS X, специалист по шрифтам и PostScript, ненавидит C++ и обожает хорошее оформление; разработчик программного обеспечения.

Jean Ghali (jghali): занимается версией программы для Windows и большей частью написания кода для управления цветом; одарённый охотник за ошибками; специалист по допечатной подготовке и инженер-программист.

Oleksandr Moskalenko (malex): сопровождает пакеты Scribus Debian и сайт Scribus, помогает с интернационализацией и переводами, является представителем GSoC; по профессии — молекулярный биолог.

Pierre Marchand (pierremarc): шрифты, холст, разметка и формирование текста. Разработчик FontMatrix. Работает разработчиком программного обеспечения в издательской компании.

Christoph Schäfer (christoph_s): создатель отчётов об ошибках и тестировщик новых технологий Scribus, редактор вики-сайта, переводчик и корректор документации, один из главных авторов официального руководства и интерактивной справки по Scribus.

Hermann Kraus (Herm): разработал подключаемый модуль Render Frame для версии 1.4, а также добавил интеграцию с GraphicsMagick и UniConvertor в версии 1.5.

Jain Basil Aliyas (jainbasil): написал фильтры импорта XTG и IDML и работает над следующим поколением Scribus Scripter.

Paul Smedley (smedles): сопровождающий версии для OS/2.
